警惕买椟还珠
#

如果你搜索Hugo博客，你会发现一个有意思的现象：建站装修博主真的非常多。

装修确实是会上瘾的，本人亲测Hugo建站没费多少功夫，但装修上三天两头修修补补，不知不觉就浑浑噩噩输出寥寥地过了一个月，实在汗颜。

怎么创建一个博客MVP（minimum viable product）
#

这篇文章整理一下我踩了巨多坑，回归理智状态后，总结的优先级最高的博客功能（个人倾向）。

如果我当时照做，应该会有一个更丝滑的开始：

首先说明，选择一个符合你需求的Hugo模板会事半功倍（否则，后期设置会很心累）：
建议选择使用人数较多的，因为Hugo建站中会有很多需要通过搜索解决的问题，而与你使用相同模板的博客文章是最直接有效的资源。
其他需要考虑的需求包括：目录、评论、tag、搜索、代码等支持、主题是否在维护中等。
目测中文博客里，Papermod和stack主题使用人数较多。

P.S. 博客主题仓库里的example site会是很好的参考资源。

  • google收录：
    小白（比如我）可能会以为只要我写了博文并po出来，就能在谷歌上搜索到了，但其实这是天大的误会……
    对于透明小站，谷歌并不一定会即时收录，也就不要提能搜索到啦。
    设置方式参考：Hugo网站如何加入Google Search Console

  • 文章tag/分类和搜索：
    通过社交媒体和搜索引擎搜索是主要的博客流量来源，而用户通常也会在发现一篇文章有用时想要探索你的其他文章。
    拥有清晰的文章tag和分类+拥有好用的搜索能力会让用户体验更好。

  • 数据统计：
    如果你不只是为了有一个电子日记本，而是也希望在这个过程中获得反馈并迭代改进，你一定需要对你的博客的浏览/用户数据有所了解。
    可以使用google analytics/Umami等方式进行后台数据统计。
    我目前使用的是google analytics，一个需要注意的坑点是记得排除本地环境调试时的浏览数据，否则数据会失真。设置方式参考：Hugo网站如何加入Google Analytics

  • 评论功能：
    对于博主本人来说，比起冷冰冰的数据，活人感更强的评论能更加激起创作热情；
    而我作为用户时，也会希望有评论区可以了解其他用户对文章质量的评价，也希望能够有渠道来对作者进行反馈。
    所以，不管是从哪个角色的角度出发，评论功能都是很必需的。

    目测目前主流的评论区方案包括：twikoo、waline、giscus（不建议非技术博客使用强制要求github账号的giscus方案）。
    我个人使用的是twikoo方案，官方教程就很清晰好上手。

其他的博客功能
#

有一个快速的丝滑的开始，有利于避免本末倒置。

不过博客作为我们的赛博小窝，也确实值得我们在过程中添砖加瓦。

收集一些我认为可以陆续增加的博客功能：

  • 图床：我使用的是cloudflare R2+piclist方案。设置方式参考：从零开始搭建你的免费图床系统和转移到 cloudflare R2 + PicList 作图床

  • 自定义字体：中文网字计划是很好的资源。设置方式参考：给博客更换字体

  • RSS

  • 设置域名：我使用的是namecheap购买域名+cloudflare CDN功能
